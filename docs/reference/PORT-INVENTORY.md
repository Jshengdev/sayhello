# Port Inventory — module-by-module (from PRD-extraction deep-readers)

## slice: doubles-harness
The Doubles codebase is a multi-agent orchestration framework for an AI digital double that generates hyper-personalized text replies while maintaining voice fidelity and safety. The harness core consists of: (1) **Orchestrator** (src/orchestrator/index.ts) — the main loop that orchestrates Thinker → Talker → Critic with regeneration up to MAX_REGEN=2, plus a Recovery Agent fallback. (2) **Critic Agent** (critic.ts + critic-prompt.ts) — a semantic scoring system across 8 axes (voice_fidelity, no_chatbot_language, register_honored, novelty_vs_recent, no_persona_break, safety, voice_texture, grounding). Verdict is "emit" only if ALL axes ≥0.7; failures trigger regen or Recovery. (3) **LLM Layer** (openrouter.ts + models.ts) — single gateway routing through OpenRouter; VOICE tier (Claude Sonnet) for user-facing Talker, FAST tier (Cerebras Qwen) for Thinker, JUDGE tier (Claude Haiku) for Critic/Recovery. (4) **Memory Layer** (embeddings.ts + memory.ts context layer) — RRF-fused entity ranking (semantic + keyword + recency) with full entity roster for grounding checks. (5) **Context Assembly** (assembler.ts + 6 layers) — parallel loading of identity, voice, memory, state, conversation, and personality into a ContextPacket. (6) **State Machine** (context builder + Moment type) — pure data substrate (Postgres) holding chat history, relational state, user patterns, concerns, recent events. (7) **Recovery Agent** (recovery.ts) — semantic fallback that either admits struggle honestly or chooses silence, never a canned string. The single most valuable export for GTM lead-story grounding is the **Critic framework**: its 8-axis rubric and verdict logic can be retargeted to score lead stories for grounding (named specifics, no fabrication), completeness (covers business fundamentals), and confidence (clear narrative arc). The LLM-call infrastructure (invocation logging, cost tracking) and the deterministic verification layer (em-dash floor, forbidden-phrase pre-check) are also port-worthy.

**env keys:** OPENROUTER_API_KEY, DATABASE_URL, FAST_LLM_PROVIDER

### context-memory-layer — `REFERENCE-ONLY` (heavy)
- path: `/Users/johnnysheng/code/doubles/src/context/layers/memory.ts`
- does: Hybrid RRF (Reciprocal Rank Fusion) entity ranking. Loads chat history (last 16 messages), recent events, and ALL shadow_entities (no pre-filter — RRF ranks them). For each competitor story: semantic similarity to current story. For each feedback item: relevance to current story's problem/solution. Weights: semantic (is this feedback about the same problem?), recency (is this feedback from a recent round?), confidence (did investor commit or just browse?). RRF ranks feedback by relevance.
